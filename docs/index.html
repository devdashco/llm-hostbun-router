llm.hostbun.cc

One OpenAI-compatible endpoint at https://llm.hostbun.cc/v1. Pick the lane by model name: local → this machine's local GPU (free), claude* → real Claude (via claude.hostbun.cc), anything else → cloud (Crazyrouter, 250+ models). No API key needed; auth is handled server-side. Docs also at docs.llm.hostbun.cc.

Quick start — one URL, pick by model

One base URL https://llm.hostbun.cc/v1 for everything. The model name picks the lane: local → the local GPU (default model); claude* → real Claude (claude.hostbun.cc); any other id → cloud. Same OpenAI API either way.

Claude lane — claudebox model: claude*

Any model whose id starts with claude is routed to claude.hostbun.cc — a claudebox (Claude Code) wrapper backed by real Claude. The gateway injects the wrapper's token, so you don't send a key. Available: claude-opus-4-8, claude-opus-4-7, claude-opus-4-6, claude-sonnet-4-6, claude-opus-4-5. Supports chat completions + streaming via the OpenAI API. (The Anthropic-native /v1/messages shape on the wrapper is currently not working — use /v1/chat/completions.)

curl https://llm.hostbun.cc/v1/chat/completions -H "Content-Type: application/json" \
  -d '{"model":"claude-opus-4-8","messages":[{"role":"user","content":"hi"}]}'

Cloud lane — Crazyrouter /v1

OpenAI-compatible aggregator (upstream api.crazyrouter.com/v1). Every request whose model isn't a local id (local / gemma / obliterated) and isn't a claude* model is proxied here with the key injected. Capabilities: chat, streaming ("stream":true), tool/function calling, structured outputs, vision, image/video/audio generation, embeddings, rerank. Full upstream reference: crazyrouter.com/docs.

# streaming
curl -N https://llm.hostbun.cc/v1/chat/completions -H "Content-Type: application/json" \
  -d '{"model":"gemini-2.5-pro","stream":true,"messages":[{"role":"user","content":"count to 5"}]}'

Structured / JSON output enforced

When you send response_format: {"type":"json_object"} (or {"type":"json_schema",…}) on /v1/chat/completions, the gateway verifies the model actually returned valid JSON before handing the reply back — it doesn't trust the model to honour the flag. This works the same on every lane (local, claude, cloud), including models that don't natively support structured outputs.

  • If the reply parses as JSON → returned unchanged.
  • If it's wrapped in a ```json … ``` fence → the fence is stripped (auto-repaired), no extra round-trip.
  • If it still isn't valid JSON → the gateway re-prompts the same model with the parse error ("your reply was not valid JSON, remake it") and retries (default 2 retries).
  • If the model never complies → you get HTTP 422 with {"error":{"code":"json_validation_failed",…}} and the last raw content, instead of a malformed body that breaks your JSON.parse.

This even works for backends that don't natively support response_format. The claude wrapper returns an empty {} if you pass it response_format, and LM Studio (local) rejects json_object. For those the gateway strips response_format and instructs the model in-prompt instead (including your JSON Schema, when given), then validates the reply the same way — so model:"claude-…" with json_schema returns exactly the shape you asked for. The cloud lane and local json_schema keep native constrained decoding.

Note: because the whole reply must be buffered to validate it, stream:true + a JSON response_format returns the content in a single reconstructed SSE chunk rather than token-by-token. Validation is structural JSON (not full JSON-Schema). Tune with env JSON_MAX_RETRIES (default 2); disable entirely with JSON_ENFORCE=0.

curl https://llm.hostbun.cc/v1/chat/completions -H "Content-Type: application/json" \
  -d '{"model":"local","response_format":{"type":"json_object"},
       "messages":[{"role":"user","content":"List 3 primary colors as a JSON array under key colors."}]}'
# → guaranteed-parseable JSON, or HTTP 422 json_validation_failed if the model refuses

Local models — select model: local or obliterated

Local requests route to LM Studio on the pbox box (RTX 4090) via llm.bofrid.dev. Private (data stays on-prem), free. Three local models:

  • local (default) → gemma-4-e4b-it-obliterated — Gemma 4 E4B abliterated/uncensored, ~8 GB, multimodal (vision) + tool calling, 32K ctx. The default-loaded model; vision works (send image_url parts incl. base64).
  • gemma → google/gemma-4-26b-a4b — 26B-A4B MoE, vision + tool calling, ~8K ctx.
  • obliterated → qwen3.6-27b-obliterated — 27B abliterated/uncensored, text-only reasoning model. Use a generous max_tokens (e.g. ≥1024) — it spends tokens on a reasoning chain before the answer. 🔒 Gated: send Authorization: Bearer <token> or it returns 401.

Drop it into any OpenAI-compatible client (opencode, Cline, Continue, the OpenAI SDK) as a custom provider — base URL https://llm.hostbun.cc/v1, API key = the obliterated token, model obliterated. /v1/models is open so the model list populates without a key; only obliterated completions need the token.

# opencode — opencode.json (or ~/.config/opencode/opencode.json)
{
  "provider": {
    "bofrid": {
      "npm": "@ai-sdk/openai-compatible",
      "options": { "baseURL": "https://llm.hostbun.cc/v1", "apiKey": "<token>" },
      "models": { "obliterated": { "name": "Qwen3.6-27B Obliterated" } }
    }
  }
}

The local models can't all fit in the 24 GB GPU (gemma and obliterated are ~19 GB each), so LM Studio JIT-swaps: the first request for a model loads it and unloads the previous one. The default local (E4B, ~8 GB) is cheap to swap back in (~5 s); switching to/from a 19 GB model is a ~20–40 s cold start. The legacy explicit path /local/v1/* (and Anthropic /local/v1/messages, native /local/api/v1/chat) still works too.

# either of these reaches the local GPU:
curl https://llm.hostbun.cc/v1/chat/completions -H "Content-Type: application/json" \
  -d '{"model":"local","messages":[{"role":"user","content":"describe"}]}'
curl https://llm.hostbun.cc/local/v1/chat/completions -H "Content-Type: application/json" \
  -d '{"model":"google/gemma-4-26b-a4b","messages":[{"role":"user","content":"describe"}]}'
