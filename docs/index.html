llm.hostbun.cc

One OpenAI-compatible endpoint at https://llm.hostbun.cc/v1. Pick the lane by model name: gemma → this machine's local GPU (free), anything else → cloud (Crazyrouter, 250+ models). No API key needed; auth is handled server-side. Docs also at docs.llm.hostbun.cc.

Quick start — one URL, pick by model

One base URL https://llm.hostbun.cc/v1 for everything. The model name picks the lane: gemma (or google/gemma-4-26b-a4b) → the local GPU; any other id → cloud. Same OpenAI API either way.

LaneSelect with modelModels
local"gemma"google/gemma-4-26b-a4b (free, on-prem, vision)
cloudany other id, e.g. "claude-sonnet-4-6"250+ (Claude, GPT, Gemini, DeepSeek, Qwen, Doubao, Grok…)
# local GPU — just select model "gemma"
curl https://llm.hostbun.cc/v1/chat/completions -H "Content-Type: application/json" \
  -d '{"model":"gemma","messages":[{"role":"user","content":"hi"}]}'
# cloud — any other model
curl https://llm.hostbun.cc/v1/chat/completions -H "Content-Type: application/json" \
  -d '{"model":"claude-sonnet-4-6","messages":[{"role":"user","content":"hi"}]}'
from openai import OpenAI            # one client; api_key is any placeholder (gateway injects it)
client = OpenAI(base_url="https://llm.hostbun.cc/v1", api_key="x")
client.chat.completions.create(model="gemma",  messages=[{"role":"user","content":"hi"}])   # local
client.chat.completions.create(model="gpt-4o", messages=[{"role":"user","content":"hi"}])   # cloud

Cloud lane — Crazyrouter /v1

OpenAI-compatible aggregator (upstream api.crazyrouter.com/v1). Every request whose model isn't the local gemma is proxied here with the key injected. Capabilities: chat, streaming ("stream":true), tool/function calling, structured outputs, vision, image/video/audio generation, embeddings, rerank. Full upstream reference: crazyrouter.com/docs.

# streaming
curl -N https://llm.hostbun.cc/v1/chat/completions -H "Content-Type: application/json" \
  -d '{"model":"gemini-2.5-pro","stream":true,"messages":[{"role":"user","content":"count to 5"}]}'

Local model — select model: gemma

Selecting gemma (or google/gemma-4-26b-a4b) on /v1 routes to LM Studio on the pbox box (RTX 4090) via llm.bofrid.dev. Private (data stays on-prem), free, fast. Pinned model google/gemma-4-26b-a4b (26B MoE, vision + tools, ~8K ctx). Vision works (send image_url parts incl. base64). The legacy explicit path /local/v1/* (and Anthropic /local/v1/messages, native /local/api/v1/chat) still works too.

# either of these reaches the local GPU:
curl https://llm.hostbun.cc/v1/chat/completions -H "Content-Type: application/json" \
  -d '{"model":"gemma","messages":[{"role":"user","content":"describe"}]}'
curl https://llm.hostbun.cc/local/v1/chat/completions -H "Content-Type: application/json" \
  -d '{"model":"google/gemma-4-26b-a4b","messages":[{"role":"user","content":"describe"}]}'

Pricing /prices.json

Actual prices (per-model discount applied, default group). Fetch programmatically:

curl https://llm.hostbun.cc/prices.json    # or /prices
# { "generated_at":"…","group_ratio":1,"count":247,
#   "models":[{"model":"claude-sonnet-4-6","type":"token",
#              "input_per_1m":1.65,"output_per_1m":8.25,"discount":0.55}, …] }

Token models: USD per 1M tokens (in/out). Image/video/audio: USD per call. Local lane (google/gemma-4-26b-a4b) = $0. Self-refreshes every 6h — last update: ….
